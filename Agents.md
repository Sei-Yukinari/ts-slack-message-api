# Agents.md

このファイルは、リポジトリに変更を加えたときに開発者が実行すべき最低限の手順を記載します。

目的
- 変更後にビルド・静的解析・テストをローカルで実行し、CIでの失敗を防ぐ。

推奨手順（変更をコミット・プッシュする前にローカルで実行）

1. 依存をインストールまたは更新

   npm install

2. ビルド（TypeScript コンパイル）

   npm run build

3. リント

   npm run lint

4. テスト

   npm test

ワークフロー
- 変更を作ったら上記コマンドを順に実行して、すべて成功することを確認してからコミット・プッシュしてください。

CI
- GitHub Actions は Node.js 24 を使用するように設定されています（.github/workflows/内を参照）。

補足
- 破壊的または大きな変更をする場合は、PR を作成して CI の結果を確認してください。
